# Contributing to CPML

Thank you for your interest in contributing to the Collaborative Perception Management Layer (CPML) project!

## How to Contribute

1. **Fork the Repository**: Create your own fork of the project
2. **Create a Branch**: Create a feature branch for your changes
3. **Make Changes**: Implement your improvements or fixes
4. **Test**: Ensure all tests pass and add new tests if needed
5. **Submit PR**: Create a pull request with a clear description

## Code Style

- Follow PEP 8 guidelines for Python code
- Use type hints where appropriate
- Write clear, descriptive docstrings
- Keep functions focused and modular

## Testing

Run tests before submitting:
```bash
pytest tests/
```

## Documentation

Update documentation for any new features or changes to existing functionality.

## Questions?

Feel free to open an issue for any questions or discussions.
